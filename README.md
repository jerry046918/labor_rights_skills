## 为什么做这个

面对公司不公正对待时，多数员工因为：

- **不懂法** —— 不知道《劳动合同法》第 87 条规定违法解除要付 2 倍经济补偿金
- **不会取证** —— 离职后才想起来保存证据，关键聊天记录、考勤、工资条早已丢失
- **怕走错棋** —— 担心"主动辞职后还能不能要补偿""签了字还能反悔吗"
- **请不起律师** —— 几千元的咨询费让人犹豫是否值得维权

而错过本应属于自己的合法权益。这个 skill 把劳动维权所需的核心知识、判例、流程模板打包到一个可被 AI 助手加载的 Skill 里，让普通员工**在遇到争议的第一时间**就能拿到一份专业级的法律分析。

## 能解决什么问题

✅ **10 类高频劳动争议全覆盖**

| # | 争议类型 | 典型场景 |
|---|---------|---------|
| 1 | 违法解除 / 违法辞退 | "业绩不达标"被口头辞退、未给补偿 |
| 2 | 拖欠工资 / 克扣工资 | 拖欠数月工资、克扣加班费 |
| 3 | 未签劳动合同（二倍工资差额） | 入职超过 1 个月仍未签书面合同 |
| 4 | 加班费争议 | 长期 996 但只发基本工资 |
| 5 | 工伤认定与待遇 | 工地受伤、公司拒绝申报工伤 |
| 6 | 调岗降薪 / 变更劳动合同 | 强行调离原岗位、单方降薪 |
| 7 | 竞业限制争议 | 离职后被起诉违反竞业协议 |
| 8 | 服务期与违约金 | 培训服务期违约金过高 |
| 9 | 经济补偿金 / 赔偿金计算 | 算不清自己该拿多少钱 |
| 10 | 社保与公积金争议 | 公司不缴/少缴社保 |

✅ **完整流程支持**
- 6 阶段调查 → 证据采集 → 检索 → 生成 → 行动建议
- 一键生成结构化《法律意见书》（8 节 Markdown 文档）
- 内置 3 个可直接填写的法律文书骨架（解除通知函 / 仲裁申请书 / 律师函）

✅ **结构化证据采集**
专门一个阶段让用户主动抛出证据——聊天截图、合同照片、工资条 PDF、谈判录音均可。文本/图片/PDF 直接由模型解析，音频走本地 FunASR 转写。所有证据统一为 evidence card（带 ★★★/★★☆/★☆☆ 可信度评级），关键事实落到 `evidence/` 目录供意见书引用。

✅ **录音证据本地处理**
谈判录音、电话录音可用本地 FunASR 转写 + 多角色分离 + 关键证据抽取。完全本地处理（音频文件不离开您的电脑），仅转写文字进入模型上下文。首次使用需安装约 3GB 依赖（VAD + ASR + 声纹 + 标点四个模型）。详见 [`references/evidence-collection.md`](./references/evidence-collection.md)。

✅ **数据可信度分级**
所有引用来源必须标注 ★★★ / ★★☆ / ★☆☆，防止道听途说的"判例"误导决策。

## 快速开始

### 安装

将本仓库克隆到您所使用的 AI 助手支持的 skills 加载路径下：

```bash
# 方式一：克隆到用户级 skills 目录（具体路径取决于您使用的 AI 助手）
git clone https://github.com/jerry046918/labor_rights_skills.git ~/skills/labor_rights_advisor

# 方式二：克隆到任意位置后通过您所用助手的 plugin / skill 加载机制引入
git clone https://github.com/jerry046918/labor_rights_skills.git
```

### 激活

在 AI 助手中直接描述您的情况即可自动激活：

> 我在上海某互联网公司工作了 3 年，月薪 2 万。上周被以"业绩不达标"为由口头辞退，没提前通知也没给补偿，怎么办？

Skill 会自动识别为「违法解除 + 经济补偿金」争议，依次完成：

1. 隐私提示与最小信息收集
2. 针对性追问（入职日期、解除形式、月工资标准等）
3. **证据采集**：主动询问用户能否提供聊天记录、合同、工资条、录音等证据，统一解析为 evidence card 落到 `evidence/` 目录
4. 法条检索（劳动法 / 劳动合同法全文）+ 类案匹配（含 WebSearch 检索最新判例）
5. 生成《法律意见书》Markdown 文档
6. 给出本周/本月可执行的行动清单 + 风险提示

## 工作原理

```
┌────────────────────────────────────────────────────────────┐
│  阶段 1：初步调查                                          │
│  收集 3 项必填：争议描述 + 用工状态 + 公司所在城市          │
└────────────────────────────────────────────────────────────┘
                            ↓
┌────────────────────────────────────────────────────────────┐
│  阶段 2：动态深化                                           │
│  根据 ① 识别争议类型（可多类并行）                          │
│  加载 references/dispute-patterns.md                       │
│  按类型生成针对性追问（必问项 + 加分项）                    │
└────────────────────────────────────────────────────────────┘
                            ↓
┌────────────────────────────────────────────────────────────┐
│  阶段 3：证据采集与解析                                     │
│  主动询问用户可提供的证据（文本/图片/PDF/音频）            │
│  加载 references/evidence-collection.md                    │
│  • 文本/图片/PDF：模型直接解析（图片/PDF 需多模态）        │
│  • 音频：后台调 scripts/process.py 走本地 FunASR 流水线    │
│  • 统一为 evidence card（含可信度评级），落到 evidence/     │
└────────────────────────────────────────────────────────────┘
                            ↓
┌────────────────────────────────────────────────────────────┐
│  阶段 4：法律依据检索                                       │
│  • 法条：从 references/labor-*-full.md 精准定位             │
│  • 案例（双轨）：                                          │
│    轨道 A — references/built-in-cases.md（10 个内置案例）   │
│    轨道 B — WebSearch 中国裁判文书网/北大法宝等权威源       │
└────────────────────────────────────────────────────────────┘
                            ↓
┌────────────────────────────────────────────────────────────┐
│  阶段 5：生成《法律意见书》                                 │
│  套用 references/opinion-template.md 八节模板               │
│  证据来自 evidence/ 下的 cards，缺失信息以【需补充】占位    │
└────────────────────────────────────────────────────────────┘
                            ↓
┌────────────────────────────────────────────────────────────┐
│  阶段 6：风险提示 + 直白行动清单                            │
│  • 维权路径（协商→监察→仲裁→诉讼）                         │
│  • 时效提醒（仲裁时效 1 年等）                              │
│  • 本周内可做的事 / 不要做的事 / 何时该找律师              │
└────────────────────────────────────────────────────────────┘
```

设计上采用**渐进式披露**——主 `SKILL.md` 约 360 行，重资料（法律全文、案例库、证据采集规则、模板）放入 `references/` 按需加载，避免每次对话都消耗大量 token。

## 可信度分级

所有法律依据必须标注以下级别：

| 标记 | 类型 | 示例 |
|------|------|------|
| ★★★ | 法律、行政法规、司法解释 | 《劳动合同法》§87、《劳动争议调解仲裁法》§27 |
| ★★☆ | 最高院指导案例、公报案例、本地区近年生效判决 | 指导案例 180 号（孙贤锋 诉 淮安西区公司） |
| ★☆☆ | 新闻报道、律师实务文章、地方人社局指导意见、学术观点 | 上海人社局 2024 年某问答 |

**关键约束：** ★☆☆ 资料不可单独支撑关键结论。若某结论仅有 ★☆☆ 依据，意见书必须明确告知"该结论需进一步核实"。

> 🔍 **帮我们一起把关**：可信度标注的准确性依赖人工核验。
> 如果你发现案例引用过时、案号失效或法条版本陈旧，欢迎[提交 Issue 或 PR](./CONTRIBUTING.md) 修正。

## 使用示例

### 场景：违法解除 + 经济补偿金

**输入：**

> 我在上海某互联网公司工作了 3 年，月薪 2 万。上周被以"业绩不达标"为由口头辞退，没提前通知也没给补偿，怎么办？

**Skill 处理：**

1. **识别**：违法解除（劳动合同法 §39、§87）+ 经济补偿金（§47）
2. **追问**：入职日期、是否签书面合同、解除通知形式、前 12 个月平均工资、是否在试用期/三期/工伤期
3. **采集证据**：用户上传微信辞退截图 + 工资流水 PDF，解析为两张 evidence card（★★☆ 截图、★★★ 银行流水）落到 `evidence/` 目录
4. **检索**：
   - 内置：§39（用人单位解除条件）、§47（经济补偿计算）、§87（违法解除 2 倍赔偿）
   - 案例：指导案例 180 号（孙贤锋 诉 淮安西区人力资源开发公司）
   - WebSearch：上海近 3 年违法解除赔偿金类案
5. **输出意见书**：含赔偿金计算公式 `3 年 × 2 万 × 2 = 12 万`、证据清单（引用 evidence card）、5 个仲裁请求项、本周可做的 5 件事

### 其他场景

skill 已通过 6 个端到端测试场景验证：违法解除、拖欠工资、未签合同、加班费、工伤、调岗降薪。

## 局限性

⚠️ **请认真阅读以下限制，避免误用：**

1. **仅适用中国大陆**——不覆盖公务员人事争议、涉外劳动争议、港澳台地区
2. **不替代律师**——意见书基于 AI 生成，复杂案件（大额索赔、竞业限制、商业秘密、刑事责任）必须咨询执业律师
3. **多模态证据需多模态模型**——图片与 PDF 证据的解析依赖模型自身的视觉能力。若所用模型不支持图像输入，skill 会跳过解析并提示用户改为提供文本摘要；音频证据不受此限制（走本地 FunASR）
4. **数据时效性**——内置法律全文为发布日版本（劳动法 2018 修正、劳动合同法 2012 修正），如遇法律修订需手动更新
5. **WebSearch 依赖**——部分案例与地方性法规依赖实时检索，网络访问受限时效果会下降
6. **金额计算精度**——意见书给出的是估算公式与数值范围，精确金额需以实际工资流水与工龄证据为准
